## 三、每个页面详细功能

### 3.1 数据仪表盘 — 数据概览

**统计卡片行（4 列）：**
| 卡片 | 指标 | 副指标 |
|------|------|--------|
| 总用户数 | 今日注册 +N | 较昨日 ↑/↓ |
| 总帖子数 | 今日发布 +N | 待审核 N 条 |
| 今日浏览量 | PV | 较昨日 |
| 今日互动 | 点赞+评论 | 活跃度 |

**图表区：**
- 近 7 日用户增长折线图
- 近 7 日帖子发布量柱状图
- 品类分布饼图（黄瓜/西红柿/土豆/水果占比）

**快捷操作：**
[待审核帖子: N条] [待认证用户: N人] [今日举报: N条]

---

### 3.2 用户管理 — 用户列表

**筛选条件：**
- 搜索框：手机号 / 昵称
- 角色筛选：全部 / 农户 / 采购商 / 司机
- 认证状态：全部 / 已认证 / 未认证
- 注册时间范围选择器

**列表字段：**
| 字段 | 说明 |
|------|------|
| ID | 用户 ID |
| 头像 | 圆形缩略图 |
| 昵称 | — |
| 手机号 | 脱敏显示 |
| 角色 | 农户/采购商/司机 标签 |
| 认证状态 | 绿色"已认证" / 橙色"未认证" |
| VIP | 是/否 金色标签 |
| 帖子数 | 点击可查看该用户全部帖子 |
| 注册时间 | YYYY-MM-DD HH:mm |
| 操作 | [详情] [编辑] [禁用/启用] |

**操作列按钮：**
- [详情]：弹出用户详情面板（个人信息+帖子列表+关注/粉丝数）
- [编辑]：可修改昵称、角色、VIP 状态
- [禁用]：二次确认弹窗 → 禁用后该用户无法发布/互动
- [启用]：解禁

**批量操作：** 勾选多条 → [批量认证] [批量封禁]

### 3.5 货源管理 — 货源帖子审核

**筛选条件：**
- 搜索框：关键词搜索帖子内容
- 类型：全部 / 供货 / 求购
- 审核状态：全部 / 待审核 / 已通过 / 已驳回
- 时间范围

**列表字段：**
| 字段 | 说明 |
|------|------|
| ID | 帖子 ID |
| 发布人 | 头像+昵称 |
| 类型 | 供货(绿) / 求购(橙) |
| 品类 | — |
| 产地 | — |
| 内容 | 文字截断 50 字，点击展开 |
| 图片数 | N 张，点击预览 |
| 发布时间 | — |
| 状态 | 待审核/已通过/已驳回 |
| 操作 | [通过] [驳回] [置顶/取消置顶] [查看详情] |

**[驳回]：** 弹窗输入驳回理由 → 推送通知给发布人
**[查看详情]：** 右侧滑出详情面板，展示完整帖子内容+图片轮播+评论列表

**批量操作：** [批量通过] [批量驳回]

### 3.8 行情管理 — 市场价格录入

**顶部品类选择器：** 下拉选择品种（黄瓜/西红柿/土豆/大白菜/夏黑无籽/巨峰葡萄/阳光玫瑰/花牛苹果）

**录入表单：**
| 字段 | 说明 |
|------|------|
| 品种 | 下拉选择 |
| 市场 | 上海辉展 / 广州江南 / 北京新发地 / +自定义 |
| 价格 | 数字输入 ¥ |
| 涨跌幅 | 自动计算或手动输入 % |
| 日期 | 日期选择器 |

**提交后** → 更新 `market_prices` 表，前端行情页面实时更新。

**批量导入：** [下载模板 CSV] [上传 CSV] — 支持 Excel 批量导入价格数据。

**历史价格列表：** 表格展示所有已录入价格，支持编辑/删除。

### 3.9 行情管理 — 涨跌数据修正

展示最近 7 天所有品种涨跌数据表格，管理员可直接点击单元格编辑涨跌幅数值（双击进入编辑模式）。

**操作：** [保存修改] → 写入数据库 + 记录操作日志。

### 3.10 商城管理 — Banner/广告管理

**列表：** Banner 名称 / 图片预览 / 跳转链接 / 排序 / 状态(启用/停用) / 操作

**操作：** [新增 Banner] 上传图片+填写标题+跳转链接 / [编辑] / [删除]

**支持：** 首页轮播 Banner、资讯横幅内容编辑。

---

### 3.11 内容管理 — 资讯公告

**列表字段：** 标题 / 类型(行情/公告/推荐) / 内容摘要 / 发布时间 / 状态 / 操作

**操作：** [新增公告] 富文本编辑器 / [编辑] / [删除] / [置顶]

发布后 → 写入 `notifications` 表全站推送，小程序首页资讯栏 + 消息 Tab 红点。

---

### 3.12 内容管理 — 推送通知

**列表：** 通知标题 / 推送范围(全部用户/指定用户/按角色) / 推送时间 / 状态

**操作：** [新建推送] → 选择推送对象 + 标题 + 内容 + 链接 → [立即发送] 或 [定时发送]

发送后 → 所有目标用户「消息」Tab 出现红点通知。

### 3.13 系统设置 — 管理员账号

**列表：** 账号 / 姓名 / 角色 / 最后登录时间 / 状态 / 操作

**操作：** [新增管理员] / [编辑角色] / [重置密码] / [禁用]

---

### 3.14 系统设置 — 角色权限

**三级角色：**

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 全部菜单 + 系统设置 + 管理员管理 |
| 运营审核员 | 货源审核 + 评论管理 + 用户认证 + 资讯公告 + 行情录入 |
| 数据查看员 | 仅查看所有数据和列表，无编辑/审核/删除权限 |

**权限矩阵表：** 每个菜单项 × 每个角色的 查看/编辑/删除/审核 权限勾选。

---

### 3.15 系统设置 — 操作日志

**筛选：** 操作人 / 操作类型 / 时间范围

**列表字段：** 操作人 / 操作类型(审核/编辑/删除/登录) / 操作对象 / 详情 / IP / 时间

所有敏感操作自动记录（帖子审核、用户封禁、价格修改、删除操作等）。

## 六、安全 + 操作规范

- 所有删除、驳回、置顶、封禁操作 → **二次确认弹窗**（标题 + 理由输入框 + 取消/确认按钮）
- 敏感操作（审核、删除、封禁、价格修改）→ 自动写入 `admin_audit_logs`
- 管理员登录 → Bearer Token 认证，2h 过期
- 登录失败 5 次/IP → 封禁 30 分钟（已有）
- 后台所有接口以 `/admin-api` 前缀（已有）

## 八、技术实现方案

**后端：** Go + Gin + GORM（复用现有架构）  
**前端：** 单页 HTML + Vanilla JS + 自定义 CSS（复用现有 admin 面板模式，升级为完整系统）  
**部署：** Go embed `internal/static/admin/`，单二进制 `/admin` 访问  
**数据库：** 复用现有 MySQL `freshbridge` 库，新增 3 张表
