# 鲜桥 FreshBridge — MVP+扩展 设计规格

> 日期：2026-05-28 | 状态：设计已确认

## 一、项目概述

水果代卖平台，对接产地农户和批发市场档口。核心链路：发布货源 → 档口接单 → 物流配送 → 市场销售 → 分账结算。

## 二、开发范围

**MVP + 扩展：**

- 核心链路：发布/接单/销售/结算
- 扩展：物流调度、微信支付分账、基础行情看板
- 2-3 个水果品类
- 农户端 + 档口端 + 司机端小程序（不做运营后台Web）

## 三、技术选型

| 层级 | 选型 | 理由 |
|------|------|------|
| 前端 | uni-app (Vue 3) | 跨端编译、市场占有率高、插件丰富 |
| 后端 | Go + Gin + GORM | 开发快、部署简单、MVP友好 |
| 数据库 | MySQL 8 + Redis | 主数据 + 缓存/Session |
| 存储 | 阿里云OSS | 图片/视频 |
| 支付 | 微信支付分账 | 合规多方分账 |
| ASR | 腾讯云/阿里云ASR | 语音转文字 |
| 地图 | 腾讯位置服务 | 微信生态适配最好 |

## 四、架构设计

### 执行策略：垂直切片（Feature Slice）

逐模块端到端交付：DB → API → 小程序 → 联调 → 下一个

### 开发顺序

| 步骤 | 模块 | 预估 |
|------|------|------|
| 1 | 项目脚手架（目录结构 + DB Schema + 小程序骨架） | 1-2天 |
| 2 | 用户认证（微信登录 + 实名认证 + 角色） | 2-3天 |
| 3 | 货源发布（发布API + 图片上传 + 语音录入 + 发布页） | 3-4天 |
| 4 | 代卖交易（接单/协议/销售录入 API + 档口端页面） | 3-4天 |
| 5 | 支付分账（微信支付分账 + 结算对账单） | 2-3天 |
| 6 | 物流调度（运单/轨迹 API + 司机端小程序） | 3-4天 |
| 7 | 行情看板（价格查询/趋势 API + 行情页面） | 1-2天 |
| 8 | 消息通知（微信订阅消息 + 站内通知） | 1-2天 |

## 五、UI 设计

### Tab 结构

| Tab | 内容 | 数据来源 |
|-----|------|---------|
| 🏠 首页 | 平台GMV、品类行情表、产区出货排行、优质档口、供需预警 | 全平台聚合 |
| 📋 货源 | 货源浏览/搜索/筛选 | 所有在线货源 |
| ＋ 发布 | 发布货源/快速记账（按角色） | — |
| 💬 消息 | 系统通知、IM消息、订阅消息 | 个人消息 |
| 👤 我的 | 产地信息、在售品种、对接档口&销售、操作入口 | 个人数据 |

### 色彩系统

| Token | 色值 | 用途 |
|-------|------|------|
| 主色 | #15803D | 按钮、导航、品牌标识 |
| 强调色 | #CA8A04 | CTA按钮、急售角标 |
| 成功/在售 | #22C55E | 正增长、在售状态 |
| 危险 | #DC2626 | 亏损、纠纷、异常 |
| 背景 | #F0FDF4 | 页面背景 |

### 交互规范

- 字号 ≥ 14px，正文 16px，金额数字 22-24px 加粗
- 触控目标 ≥ 44px，间距 ≥ 8px
- 语音入口占页面 1/3，适用于嘈杂环境
- 关键操作按钮固定底部，min-height 48px
- 离线缓存 + 联网自动同步

### 模块跳转关系

所有操作从「我的」→ 跳转独立页面：
- 上架货源 → 发布页（语音+表单+拍照）
- 档口对接 → 档口列表/详情页
- 销售看板 → 实时数据页
- 对账结算 → 结算管理页
- 物流追踪 → 运单追踪页
- 我的货源 → 货源列表管理页

## 六、数据模型（核心表）

```
users           — 用户/认证
products        — 货源
trade_orders    — 代卖订单/协议
sales_records   — 销售记录（档口上报）
settlements     — 结算记录
logistics       — 物流运单
market_prices   — 行情数据
```

## 七、API 模块

- `POST /api/auth/wechat-login` — 微信登录
- `POST /api/products` — 发布货源
- `GET /api/products` — 货源列表/搜索
- `POST /api/trades` — 发起代卖
- `POST /api/sales` — 录入销售（语音/手动）
- `GET /api/settlements` — 对账单
- `POST /api/logistics` — 发布运单
- `GET /api/market/prices` — 行情查询
